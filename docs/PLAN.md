# StillHungry — 미래 개발 계획

---

## 사용자 로그인 & 즐겨찾기

### 배경
현재 사용자 페이지는 로그인 없이 단지 → 식당 → 메뉴 순으로 탐색하는 구조.
자주 가는 식당이 정해져 있는 사용자라면 매번 탐색하는 게 번거로움.
로그인 후 즐겨찾기한 식당들의 오늘 메뉴를 한 화면에서 바로 볼 수 있게 한다.

### 데이터 모델
- `User` 테이블 추가
  - 소셜 로그인(카카오 등) 또는 이메일/비밀번호
- `Favorite` 테이블 추가
  - userId, cafeteriaId (사용자 ↔ 식당 N:M)

### 기능

#### 사용자 인증
- 소셜 로그인 (카카오, 구글) 또는 간단한 이메일 가입
- JWT 기반 인증 (어드민과 별도 토큰 체계)

#### 즐겨찾기
- 식당 상세 페이지에서 즐겨찾기 추가/해제 (하트 버튼)
- 로그인하지 않은 경우 → 로그인 유도

#### 즐겨찾기 대시보드 (홈 화면 대체)
- 로그인한 사용자의 홈(`/`) → 즐겨찾기 식당들의 오늘 메뉴 카드 목록
- 각 카드: 식당 이름 + 조/중/석식 메뉴 요약 (주메뉴 위주)
- 메뉴가 아직 없는 식당은 "오늘 메뉴 미등록" 표시
- 비로그인 사용자는 기존 단지 탐색 화면 유지

### UX 흐름
1. 사용자 로그인
2. 식당 페이지에서 ♡ 버튼으로 즐겨찾기 등록
3. 홈 화면 → 즐겨찾기 식당들의 오늘 메뉴 한눈에 확인
4. 카드 클릭 → 해당 식당 상세로 이동

## 메뉴 사진 업로드

### 배경
매일 메뉴판을 사진으로 찍어서 올리면 텍스트 입력 부담을 줄일 수 있음.
텍스트 메뉴와 병행해서 사진도 함께 제공하면 사용자 경험이 향상됨.

### 데이터 모델
- `Menu` 모델에 `imageUrl String?` 필드 추가

### 스토리지
- Supabase Storage 활용 (기존 DB와 같은 플랫폼)
- 업로드 흐름: 클라이언트 → 서버 → Supabase Storage → imageUrl 저장

### 기능
- 어드민 메뉴 등록/수정 시 사진 업로드 (선택사항)
- 사용자 화면: 사진 있으면 메뉴 카드에 표시, 없으면 기존 텍스트만 표시

## 메뉴 템플릿 기능

### 배경
식당들이 새로운 메뉴도 나오지만, 동일한 메뉴를 순환해서 제공하는 경우가 많음.
매번 메뉴를 새로 입력하는 대신, 미리 등록해둔 메뉴를 선택해서 빠르게 등록할 수 있게 한다.

### 데이터 모델
- `MenuTemplate` 테이블 추가
  - 식당(cafeteriaId), 식사유형(mealType), 이름, 메뉴 항목 목록
- `MenuTemplateItem` 테이블 추가
  - 템플릿에 속한 개별 메뉴 항목 (이름, kcal, isMain)

### 어드민 기능
- 식당 설정 탭에 "메뉴 템플릿 관리" 섹션 추가
- 템플릿 등록 / 수정 / 삭제
- 메뉴 등록 시 "템플릿에서 불러오기" 버튼 → 선택하면 메뉴 항목 자동 채움
  - 불러온 뒤 항목 추가/삭제 등 자유롭게 편집 가능

### UX 흐름
1. 어드민이 자주 쓰는 메뉴 조합을 템플릿으로 저장
2. 메뉴 등록 폼 → "템플릿 불러오기" 클릭 → 해당 식사유형 템플릿 목록 표시
3. 선택 → 항목 자동 입력 (이후 수동 수정 가능)

## ✅ 식당 위치 & 지도 보기 (완료)

### 배경
식당이 정확히 어디 있는지 주소 텍스트만으로는 파악하기 어려움.
지도에서 핀으로 위치를 보여주면 처음 방문하는 사용자도 쉽게 찾아갈 수 있음.

### 데이터 모델
- `Cafeteria` 모델에 `latitude Float?`, `longitude Float?` 필드 추가
- 어드민에서 지도 클릭 또는 주소 검색으로 좌표 저장

### 기능
- 어드민: 식당 설정에서 위치 핀 지정 (카카오맵 or 구글맵 API)
- 사용자: 식당 상세 페이지에 미니 지도 표시 (핀 + 주소)
- 단지 내 식당 목록 페이지에서 "지도 보기" 모드 전환 가능

## 내 위치 기반 가까운 식당 추천

### 배경
단지나 건물을 모르는 신규 사용자도 현재 위치 기준으로 가까운 식당을 바로 찾고 싶어함.
위치 권한을 허용하면 거리 순으로 정렬된 식당 목록을 제공한다.

### 기능
- 홈 탐색 탭에 "내 근처 식당" 섹션 추가
- 브라우저 Geolocation API로 현재 좌표 획득
- 서버에서 좌표 기반 거리 계산 후 가까운 식당 N개 반환 (Haversine 공식)
- 각 식당 카드에 거리 표시 (예: 도보 3분 · 230m)
- 위치 권한 거부 시 안내 문구 표시

### API
- `GET /public/cafeterias/nearby?lat=&lng=&limit=` 엔드포인트 추가

## 별점 & 리뷰 기능

### 배경
메뉴 정보 외에 실제 이용자들의 평가를 보고 식당을 선택하고 싶어하는 수요가 있음.
간단한 별점과 한 줄 리뷰로 식당의 전반적인 만족도를 공유할 수 있게 한다.

### 데이터 모델
- `Review` 테이블 추가
  - userId, cafeteriaId, rating (1~5), content, createdAt
- `Cafeteria` 모델에 평균 평점 캐시 필드 추가 (또는 집계 쿼리로 실시간 계산)

### 기능

#### 사용자
- 식당 상세 페이지 하단에 별점(★) 선택 + 리뷰 텍스트 입력
- 로그인 사용자만 리뷰 작성 가능 (비로그인 시 로그인 유도)
- 내가 쓴 리뷰 수정 / 삭제
- 식당별 전체 리뷰 목록 & 평균 별점 표시

#### 어드민
- 신고된 리뷰 블라인드 처리 (슈퍼어드민)

## ✅ 한 주 메뉴 한눈에 보기 (완료)

### 배경
오늘 메뉴만 보는 게 아니라 이번 주 전체 메뉴를 미리 파악하고 싶어하는 사용자가 있음.
주간 캘린더 뷰로 날짜별 메뉴를 한 화면에 보여준다.

### 기능
- 식당 상세 페이지에 "주간 보기" 탭 추가
- 월~일 7일을 열(column)로, 조/중/석식을 행(row)으로 배치한 테이블 형태
- 오늘 날짜 컬럼 하이라이트
- 메뉴가 없는 날은 "미등록" 표시
- 좌우 스와이프로 주 단위 이동

### API
- `GET /public/cafeterias/:id/menus?from=&to=` 날짜 범위 파라미터 활용 (기존 API 확장)

## 카테고리별 메뉴 필터

### 배경
특정 메뉴(예: 제육볶음)가 먹고 싶을 때 어느 식당에서 나오는지 바로 찾고 싶어함.
메뉴 항목 키워드 기반으로 해당 메뉴가 등록된 식당을 필터링해서 보여준다.

### 기능
- 홈 탐색 탭에 키워드 검색창 (현재 식당명 검색 → 메뉴명 검색으로 확장)
- 오늘 기준으로 해당 키워드가 포함된 메뉴를 제공하는 식당 목록 표시
- 식당 카드에 매칭된 메뉴 항목 하이라이트
- 자주 검색되는 카테고리 태그 버튼 제공 (예: 제육, 돈까스, 비빔밥, 국수 등)

### API
- `GET /public/menus/search?q=&date=` 엔드포인트 추가
  - 메뉴 항목 이름에 키워드 포함된 식당 목록 반환
